# Документация и аналитический разбор Sentinel-Path

`Sentinel-Path` — это аналитический инструмент для оценки структурной хрупкости и рисков в проектных графиках. Данный документ описывает архитектуру, логику обработки данных и планы по развитию проекта.

---

## Архитектура проекта

### Таблица компонентов

| Файл | Роль в проекте | Зависимости |
| :--- | :--- | :--- |
| `sentinel_path/__main__.py` | Точка входа CLI, оркестрация загрузки данных и вызова анализа. | `argparse`, `json`, `pathlib`, `sentinel_path.engine`, `sentinel_path.path_handler` |
| `engine.py` | Фасад-оркестратор, объединяющий топологический, стохастический анализ и визуализацию. | `matplotlib`, `numpy`, `core.fragility`, `core.topology`, `models.schemas`, `stochastic.simulation` |
| `sentinel_path/path_handler.py` | Фильтр на входе: очистка путей и проверка прав доступа перед анализом. | `os`, `pathlib` |
| `core/topology.py` | Построение графа (DAG) и расчет критического пути (CPM). | `networkx`, `models.schemas` |
| `core/fragility.py` | Анализ точек хрупкости и расчет индекса сходимости путей (PCI). | `models.schemas` |
| `stochastic/simulation.py` | Симуляция Монте-Карло с распределением Beta-PERT и расчет чувствительности. | `numpy`, `scipy`, `models.schemas` |
| `models/schemas.py` | Определение контрактов данных и валидация через Pydantic. | `pydantic` |

### Поток данных (Data Flow)

Пайплайн обработки данных в проекте организован следующим образом:

1.  **Ввод данных**: Пользователь передает пути к JSON-файлам через CLI (например, `--tasks tasks.json`).
2.  **Предварительная обработка**: Модуль `sentinel_path/path_handler.py` нормализует пути (решая проблемы со слешами Windows/Linux) и проверяет права доступа через `os.access`.
3.  **Валидация схем**: Сырые JSON-данные преобразуются в типизированные модели Pydantic в `engine.py`.
4.  **Топологический анализ**:
    -   Строится направленный ациклический граф (DAG).
    -   Выполняется расчет ранних и поздних сроков (CPM) в `core/topology.py`.
5.  **Анализ рисков**:
    -   **Хрупкость**: `core/fragility.py` выявляет узлы, где сходятся критические пути.
    -   **Стохастика**: `stochastic/simulation.py` запускает тысячи итераций для оценки вероятности завершения проекта.
6.  **Вывод**: Результаты упаковываются в `SentinelReport` и сохраняются в JSON или визуализируются в виде графиков.

## Инструкции по запуску

Для запуска анализа из терминала используйте следующую команду:

```bash
python -m sentinel_path --tasks tasks.json --dependencies deps.json --output report.json
```

Если вам необходима визуализация рисков (гистограммы и S-кривые), добавьте путь к директории для графиков:

```bash
python -m sentinel_path --tasks tasks.json --dependencies deps.json --charts-dir ./artifacts/charts
```

## Решение проблем (Troubleshooting)

| Ошибка | Причина | Решение |
| :--- | :--- | :--- |
| `CycleError` | В графе зависимостей обнаружена петля (цикл). | Проверьте `deps.json`: задача не может зависеть от самой себя или образовывать круг. |
| `Path is not accessible` | У программы нет прав на чтение файла или путь указан неверно. | Проверьте существование файла и права доступа в вашей ОС. |
| `ValidationError` | Входной JSON не соответствует ожидаемой структуре (например, пропущено поле `duration`). | Сверьте формат файлов с описанием в `models/schemas.py`. |
| `GraphTopologyError` | Обнаружены изолированные задачи, не связанные с общим графиком. | Убедитесь, что каждая задача имеет связь (кроме стартовых/финишных). |

## Инвентаризация зависимостей

Для работы проекта необходимы следующие библиотеки (указаны в `pyproject.toml`):

-   `networkx`: Математическое представление графа проекта и проверка его связности.
-   `numpy`: Векторизованные вычисления для быстрой работы симуляции Монте-Карло.
-   `pydantic`: Гарантия целостности данных на входе и выходе системы.
-   `scipy`: Статистический анализ (коэффициент Спирмена) для оценки влияния задач на сроки.
-   `matplotlib`: Генерация визуальных отчетов в формате PNG.
